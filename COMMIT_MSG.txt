Fix: add trailing slash to frontend orders API calls

Frontend was calling /api/v1/orders (no trailing slash), but the route
is @router.post("/") which requires a trailing slash. FastAPI's default
redirect_slashes=True sends a 307 redirect, which browsers handle
inconsistently for CORS-preflighted POST requests — the redirect URL
may lose the Origin header, causing CORS to fail silently.

Fix: Added trailing slash to both /orders/ (create) and /orders/?
(list) endpoints in the frontend API client, so requests hit the
correct route directly with no redirect.
